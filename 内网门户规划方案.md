# 内网门户规划方案

## 一、项目概述
构建一个统一的内网门户，整合5-15个内部系统入口，提供便捷的导航和搜索功能，支持移动端访问。

## 二、技术选型

### 前端技术栈
- **框架**: Vue 3 + Vite
  - 选择理由：轻量、易上手、生态完善，适合中小型项目
  - 组件库：Element Plus 或 Ant Design Vue
- **移动端适配**: 响应式设计 + 移动端优先策略
- **状态管理**: Pinia
- **构建工具**: Vite

### 后端技术栈
- **框架**: Node.js + Express
  - 选择理由：与前端技术栈统一，开发效率高
- **数据存储**: JSON文件 或 SQLite（轻量级）
  - 初期可用JSON配置文件管理系统链接
- **API设计**: RESTful API

### 部署环境
- **服务器**: Windows Server
- **Web服务器**: IIS 或 Nginx for Windows
- **运行时**: Node.js LTS版本

## 四、功能模块设计

### 1. 系统链接聚合
- 分类展示（按部门、功能类型等）
- 卡片式/列表式布局切换
- 链接图标、名称、简介
- 点击跳转到对应系统

### 2. 搜索功能
- 实时搜索（输入即搜索）
- 搜索历史记录
- 热门搜索推荐
- 模糊匹配（名称、描述、标签）

### 3. 移动端适配
- 响应式布局，自适应不同屏幕
- 触摸友好的交互设计
- 移动端优化的导航菜单
- 支持添加到手机桌面（PWA）

### 4. 管理后台（可选）
- 系统链接的增删改查
- 分类管理
- 用户访问统计

## 七、部署方案

### 开发环境
- 本地开发：`npm run dev`
- 热重载开发服务器

### 生产环境
1. **构建前端**: `npm run build`
2. **部署文件**:
   - 将dist目录部署到IIS/Nginx
   - 将后端文件部署到Node.js环境
3. **配置IIS/Nginx**:
   - 反向代理API请求到Node.js
   - 静态文件服务
   - HTTPS配置（推荐）

### IIS配置示例
```xml
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="Node.js API" stopProcessing="true">
          <match url="^api/.*" />
          <action type="Rewrite" url="http://localhost:3000/{R:0}" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>
```
